# 精读提问清单 —— 通用骨架

读每节时对照这张**领域无关**的骨架,核对作者交代清楚没有;没交代的,往往是这篇的
模糊点或局限,也可能是你课题的切入空白。**方向专属的问题**见所选速查表末尾的
"该方向追问项"(`references/domains/*.md`)。两者配合用。

## 1. 问题与动机
- 到底要解决什么问题?现有方法差在哪(作者明确指出的 gap)?
- 为什么这个问题重要 / 现在才能做?

## 2. 方法
- 核心做法一句话是什么?相对前人**新在哪**(1–3 条,具体可证伪)?
- 关键模块 / 公式 / 设计选择,动机是什么、替代了什么旧做法?
- 是新方法、新训练目标、新数据/任务,还是新分析?(别被包装词迷惑)

## 3. 数据与实验
- 用什么数据(规模、来源、是否公开)?仿真还是真实?
- 跟哪些基线比?是否同等设置下公平比较?
- 用什么指标?消融实验拆了哪些组件、各掉多少分?

## 4. 结果可信度
- 提升来自方法本身,还是更大模型/更多数据/更多算力?
- 有显著性 / 多随机种子 / 方差报告吗?
- 有没有失败案例、负面结果、诚实的局限讨论?

## 5. 泛化与代价
- 分布外 / 跨场景 / 跨数据集表现?
- 计算成本、训练/推理开销、可复现性(代码与数据可得吗)?

## 用法
读完一节,看它回答了骨架 + 方向追问项里的哪几条。全文读完仍"未提及"的条目,
连起来就是这篇的边界,记进精读小结的"局限/空白"。
