# 三遍法(Keshav's three-pass approach)精简版

读论文不要从第一句逐字读到最后。分三遍,每遍目的不同,前两遍就够"快速读懂"。

## 第一遍 · 鸟瞰(目标 5 分钟)
只读:标题 → 摘要 → 引言(尤其末段的"本文贡献"列表) → 各节小标题 →
结论 → 扫一眼图表标题与参考文献里你认识的名字。
读完要能回答"五个 C":
- Category:什么类型(新方法 / 综述 / 分析 / 系统)
- Context:基于哪些前人工作,用什么理论基础
- Correctness:假设看起来站得住吗
- Contributions:核心贡献是什么
- Clarity:写得清楚吗
判断:**值不值得读第二遍**。不值就停。

## 第二遍 · 抓内容(目标 ≤1 小时)
通读正文,但**跳过证明、繁琐推导**。看懂图表(它们常是结论所在)。
标记读不懂、想回头查的参考文献。目标:能向别人**复述这篇做了什么、结论是什么**,
但不要求能自己复现。陪读时绝大多数论文停在这一遍就够。

## 第三遍 · 吃透(只在需要复现/深究时)
"虚拟重做"这篇:跟着作者的假设一步步重建,挑战每个推导和实验设计,
想"换我会怎么做"。读完能凭记忆重构全文结构,并指出隐含假设与潜在问题。

## 陪读时怎么用
- 默认带用户走 第一遍 → 第二遍。
- 第二遍按"节"切片,一节一停。
- 只有用户对某节明确想钻细节,才对那一节启动第三遍。
